# Architecture Decision Records (ADR)

Short, dated records of significant technical decisions. The point is to
capture **why** a decision was made — not what the code does (the code
explains itself, the reasoning doesn't).

Format (per Michael Nygard):
- Title (one line, status in brackets)
- Context — what's the situation that demanded a decision
- Decision — what was chosen
- Consequences — what that costs / unlocks

When you make a decision worth recording, copy `0000-template.md` and
number the file. Don't edit accepted ADRs — supersede them with a new
ADR that references the old one.
